# PaperPlotter 执行计划

## 当前阶段

- 阶段：第 8 步，Z 单点标定向导。
- 当前目标：实现点粗找和线细调流程，生成对应测试 G-code，并保存 basePenDownZ。
- 已通过验收：第 1、2、3、4、5、6、7 步已完成；浏览器验证确认 UI 显示醒目的 `G92 X0 Y0` 归零提示，G-code 文件头测试覆盖同样提示。
- 下一验收标准：用户可选择测试点；阶段 A 生成下降一步并点一下的 G-code；阶段 B 生成 3-4 条候选短线；用户选择最佳线后保存 basePenDownZ。

## 分阶段任务

1. 项目骨架（已完成）
   - 建立 Vite/React/TypeScript 基础配置。
   - 建立 `src/types`、`src/state`、`src/features` 的目录边界。
   - 实现默认 ProjectFile 状态和基础 App 布局。
   - 添加 Vitest 测试环境。

2. SVG 纸张画布（已完成）
   - 用毫米坐标绘制纸张矩形和网格。
   - 支持 A4/A5/B5/自定义尺寸和横纵向切换。
   - 支持基础缩放和平移。

3. 测试路径对象（已完成）
   - 添加矩形、十字、网格测试图案。
   - 实现对象级拖动和路径预览。

4. 基础 G-code 导出（已完成）
   - 实现路径到 G-code 的转换。
   - 暴露抬笔、落笔、速度、Z 轴参数。
   - 实现导出前安全检查。

5. 照片导入与纸张四角标定（已完成）
   - 导入照片作为标定背景。
   - 点选四角并计算图片到纸面透视变换。
   - 显示校正后的纸面背景。

6. 机器参考线标定（已完成）
   - 点选参考线两端。
   - 选择参考线代表 X 或 Y 方向。
   - 计算纸张到机器坐标映射。

7. 归零提示流程（已完成）
   - 导出前显示 `G92 X0 Y0` 的明确操作提示。
   - 把提示写入 G-code 注释头。

8. Z 单点标定向导
   - 阶段 A：点粗找。
   - 阶段 B：线细调。
   - 保存 `basePenDownZ` 并用于后续导出。

9. 文本对象与假字体
   - 实现文本对象类型和假字体 provider。
   - 把简单字符编译为 stroke paths。
   - 支持预览和导出。

10. 第一种中文笔画字库
    - 在坐标/G-code 闭环稳定后接入中文 stroke provider。
    - 验收“你好”“实验报告”等中文测试内容。

## 当前风险

- 第 10 步涉及中文笔画数据源，必须等前 1-9 步闭环稳定后再接入，避免字体数据复杂度干扰坐标与 G-code 主线。
- 本 MVP 不做串口发送，验收机器执行效果时需要用户使用外部 sender。
