# PaperPlotter 执行计划

## 当前阶段

- 阶段：软件侧 MVP 收口完成，正在补齐公开展示与实机验收入口。
- 当前目标：补充 `README.md`，说明项目用途、本地启动、云端部署和实机验收流程；同时已通过 GitHub CLI 写入仓库 Description，随后启动本地开发服务器供用户浏览器实操验收。
- 已通过验收：第 1 至第 10 步的软件功能均已完成；类型检查、单元/组件测试、生产构建和浏览器回归均通过。
- 下一验收标准：用户打开本地开发地址后，能按 README 和页面提示完成一次导出前检查；真实写字机侧仍需按 `G92 X0 Y0` 流程运行导出的测试路径或中文示例 G-code。

## 分阶段任务

1. 项目骨架（已完成）
   - 建立 Vite/React/TypeScript 基础配置。
   - 建立 `src/types`、`src/state`、`src/features` 的目录边界。
   - 实现默认 ProjectFile 状态和基础 App 布局。
   - 添加 Vitest 测试环境。

2. SVG 纸张画布（已完成）
   - 用毫米坐标绘制纸张矩形和网格。
   - 支持 A4/A5/B5/自定义尺寸和横纵向切换。
   - 支持基础缩放和平移。

3. 测试路径对象（已完成）
   - 添加矩形、十字、网格测试图案。
   - 实现对象级拖动和路径预览。

4. 基础 G-code 导出（已完成）
   - 实现路径到 G-code 的转换。
   - 暴露抬笔、落笔、速度、Z 轴参数。
   - 实现导出前安全检查。

5. 照片导入与纸张四角标定（已完成）
   - 导入照片作为标定背景。
   - 点选四角并计算图片到纸面透视变换。
   - 显示校正后的纸面背景。

6. 机器参考线标定（已完成）
   - 点选参考线两端。
   - 选择参考线代表 X 或 Y 方向。
   - 计算纸张到机器坐标映射。

7. 归零提示流程（已完成）
   - 导出前显示 `G92 X0 Y0` 的明确操作提示。
   - 把提示写入 G-code 注释头。

8. Z 单点标定向导（已完成）
   - 阶段 A：点粗找。
   - 阶段 B：线细调。
   - 保存 `basePenDownZ` 并用于后续导出。

9. 文本对象与假字体（已完成）
   - 实现文本对象类型和假字体 provider。
   - 把简单字符编译为 stroke paths。
   - 支持预览和导出。

10. 第一种中文笔画字库
    - 在坐标/G-code 闭环稳定后接入中文 stroke provider。
    - 验收“你好”“实验报告”等中文测试内容。
    - 当前状态：已完成。内置 `BasicChineseStrokeFontProvider` 覆盖“你好”“实验报告”，浏览器验证“实验报告”生成 23 条中文 polyline，并与默认方框一起形成 24 条可导出路径。

11. MVP 实机验收（进行中）
    - 软件侧已能导入照片、点选四角、点选机器参考线、放置测试路径/文本、导出 G-code、提示 `G92 X0 Y0`、执行单点 Z 标定、配置关键机器参数。
    - 仍需用户用真实写字机和外部 sender 验证：G92 后输出位置是否与纸面预览一致；Z 细调候选线是否能选出稳定落笔高度。

12. README 与云端展示信息（进行中）
    - 新增 `README.md`，面向 GitHub 访问者说明项目定位、核心功能、技术栈、启动方式、构建方式、云端部署方式和实机验收步骤。
    - 通过 GitHub CLI 写入仓库 Description，便于仓库页面展示和搜索。
    - 本阶段完成后启动本地 Vite 开发服务器，交给用户浏览器实操验收。

## 当前风险

- 当前最大风险已从“软件实现缺口”转为“实机误差验证”：机器皮带松紧、笔夹高度、纸张固定方式、sender 的坐标模式都可能影响最终落纸位置。
- 本 MVP 不做串口发送，最终验收必须由用户使用外部 sender 执行导出的 `.gcode` 文件。
- README 和本地浏览器验收只能证明项目可安装、可启动、可导出，不能替代真实机器落纸位置验收。
