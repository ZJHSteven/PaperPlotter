# PaperPlotter 执行计划

## 当前阶段

- 阶段：软件侧 MVP 已进入“参考图级 UI 复刻与实机前交互补强”。
- 当前目标：按 `docs/assets/ui-reference-tool-workbench.png` 完成首轮工具台密度和结构复刻，继续等待用户用真实写字机验证 Web Serial Z 标定、G92 原点和导出 G-code 落纸位置。
- 已通过验收：第 1 至第 10 步的软件功能均已完成；上一轮 typecheck、test、build 均通过；IndexedDB 图片存储和 Web Serial Z 标定入口已落地。

## 分阶段任务

3. 测试路径对象（已完成）

4. 基础 G-code 导出（已完成）

5. 照片导入与纸张四角标定（已完成）

## 当前风险

- 当前最大风险已从“软件实现缺口”转为“实机误差验证”：机器皮带松紧、笔夹高度、纸张固定方式、sender 的坐标模式都可能影响最终落纸位置。
- 本 MVP 不做串口发送，最终验收必须由用户使用外部 sender 执行导出的 `.gcode` 文件。
- README 和本地浏览器验收只能证明项目可安装、可启动、可导出，不能替代真实机器落纸位置验收。
- Web Serial 需要浏览器支持和用户授权选择串口；在不支持 Web Serial 的浏览器里仍需保留“复制 G-code 到外部 sender”的后备流程。
- 当前浏览器 UI 已通过自动化验证：默认 1680×944 视口无页面滚动、左右栏同高、无框架报错；剩余风险集中在 Web Serial 授权弹窗、串口占用和真实机器运动误差。
